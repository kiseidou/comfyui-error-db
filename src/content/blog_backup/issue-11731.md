## エラーの概要

この記事は、ComfyUIを使用しているユーザーが「Portable版Windows」で『Save (API format)』ボタンが表示されない問題について記述します。

## 原因

- **Python環境の不備**：
  - ポータブルバージョンのインストールでは、必要なライブラリや設定が不足している可能性がある。
- **カスタムノードの競合**：
  - カスタムノードが問題を引き起こす場合もある。しかし、カスタムノードを無効にしても問題は解決しない。

## 解決策

1. **Python環境の再構成**：

   ポータブルバージョンでのインストール後にPython環境の設定が正しく行われていない可能性があります。以下の手順でPython環境を再構成します。

   1. ComfyUIのディレクトリ内にある`requirements.txt`ファイルを確認し、必要な依存関係をリストアップします。
   2. Pythonインタプリタを使用して依存関係をインストールします。コマンドラインで以下のコマンドを実行してください。

      ```bash
      pip install -r ComfyUI/requirements.txt
      ```

2. **カスタムノードの確認**：

   カスタムノードが問題を引き起こす場合もありますので、その可能性を検討します。
   
   1. `comfyui`ディレクトリ内のカスタムノードフォルダ(`custom_nodes`)を削除または移動して試みてください。

3. **ComfyUI設定の確認**：

   `comfyui.yaml`ファイルに問題がないか確認し、必要であれば初期設定に戻すことも検討してください。
   
4. **バージョンの再インストール**：
   - ComfyUIとフロントエンドを最新版に更新してから再度インストールします。

## まとめ

ポータブル版Windows上でComfyUIで『Save (API format)』ボタンが表示されない問題は、Python環境やカスタムノードの競合により発生することが多いです。上記の手順を参考にすることで、エラーが解決する可能性が高いです。

それでも解決しない場合は、公式Discordサーバーで助けを求めると良いでしょう。
