## エラーの概要
バージョン0.4.0以降、ComfyUIが正常に動作しなくなったと報告されています。カスタムノードの再インストールやPython環境のクリーンアップを行っても改善せず、最新版の更新ができていない状況です。

## 原因
1. **Python環境の不整合**：過去のバージョンからの引き継ぎで、Pythonの仮想環境（`.venv`）に問題が生じている可能性があります。
2. **カスタムノードの互換性問題**：更新版と非対応のカスタムノードを使用している場合、エラーが発生します。

## 解決策
### Python環境をクリーンアップする

1. ComfyUIのインストールディレクトリ内の全ての`.venv`ディレクトリを削除。
2. 標準Python環境のクリーンアップ：
   ```bash
   python -m venv .venv
   ```
3. ComfyUIを最新版に再インストール。

### カスタムノードの互換性確認と更新

1. 公式ドキュメントやコミュニティフォーラムでカスタムノードの最新バージョンを調査。
2. 非対応なカスタムノードは置き換えまたは削除。

```bash
# カスタムノードディレクトリを確認
cd custom_nodes/
ls

# 必要に応じて、非対応のカスタムノードを削除
rm -rf non_compatible_node_folder/

# 最新版のカスタムノードをインストールまたは更新
pip install --upgrade https://github.com/username/repository/archive/master.zip
```

### 完全な再インストール手順

1. 既存の全てのファイルやフォルダ（テンプレート、入力ディレクトリ、出力ディレクトリ等）をバックアップ。
2. Python仮想環境`.venv`とComfyUI本体を完全に削除。
3. バックアップから必要なテンプレートやカスタムノードを復元。
4. ComfyUIを最新版で再インストール。

## まとめ
Python環境の不整合と古いカスタムノードが原因で、ComfyUIが正常に動作しない場合があります。これらの問題を解決するには、Python環境をクリーンアップし、カスタムノードの互換性を確認・更新することが必要です。
