## エラーの概要

ComfyUIバージョンv0.8.2で、`CLIPLoader (CLIローダー)` ノードを使用中に 'NoneType' object has no attribute 'Params' というエラーメッセージが表示されます。

このエラーは特定のワークフロー（特に `wan2.2_ti2v_5B` テンプレートを実行する場合）で起きることが多いです。

## 原因

このエラーは以下の状況で発生します：

- カスタムノードが適切にロードされていない。
- Pythonの依存ライブラリが壊れている（古いバージョンや欠落している）。
- ComfyUI本体のコードが古く、最新のカスタムノードと互換性がない。

## 解決策

### 1. カスタムノードを無効にする

まず、カスタムノードを一度すべて無効にしてみます。具体的な手順は以下の通りです：

- `Settings (設定)` メニューを開き、`Disable Custom Nodes (カスタムノード無効化)` を選択。
- 再起動して確認。

### 2. Pythonの依存ライブラリを更新する

ComfyUIのPython環境で必要なライブラリが最新であることを確認します。以下は更新手順です：

1. 管理者権限でコマンドプロンプトを開き、以下のコマンドを実行：
   ```bash
   pip install --upgrade -r requirements.txt
   ```
2. 再起動してエラーが解決したか確認。

### 3. ComfyUIの最新バージョンに更新する

ComfyUIのソースコード自体が古い可能性もあります。以下の手順で更新を行います：

1. [公式GitHubリポジトリ](https://github.com/ComfyUI)から最新のソースを取得。
2. コピーした新しいソースコードを元のインストールディレクトリに上書きして再起動。

## まとめ

`CLIPLoader 'NoneType' object has no attribute 'Params'` エラーは、カスタムノードや依存ライブラリの問題が原因で発生します。まずはこれらの要素を無効化したり更新したりして解決を試みてください。それでもエラーが続く場合は、ComfyUI自体のバージョンアップデートが必要かもしれません。
