## エラーの概要
"ComfyUI" を使用中に、エラーメッセージが表示され、ワークフローテンプレートのアップデートに失敗することがあります。

## 原因
### 1. パッケージ依存関係の更新漏れ
`requirements.txt` で定義された `comfyui-workflow-templates==0.8.4` への更新が完了していない場合、エラーが発生します。
```python
- comfyui-workflow-templates==0.8.0
+ comfyui-workflow-templates==0.8.4
```

### 2. カスタムノードの互換性問題
このバージョンでは一部テンプレートがローカルユーザー向けにカスタムノードを要求するものがあり、これがインストールされていない場合や非対応な場合、エラーになります。

## 解決策

### 1. Python 環境の更新
依存関係を最新のバージョンに更新します。
```bash
pip install --upgrade comfyui-workflow-templates==0.8.4
```

### 2. カスタムノードの確認とインストール
テンプレートが要求するカスタムノードをインストールします。具体的なノード名は、エラーメッセージやテンプレート設定で確認してください。

### 3. デフォルトテンプレートの更新手順
- **Templates Manager (テンプレートマネージャ) を開く**:
  - 更新可能なテンプレートが表示されるはずです。
  
- **Update Workflow Templates (ワークフローテンプレート更新)**:
  - "Update workflow templates to v0.8.4" のオプションを選択し、実行します。

## まとめ
エラー 'Update workflow templates to v0.8.4' を解決するには、Pythonの依存関係を最新に更新し、必要であればカスタムノードもインストールしてください。これによりワークフローテンプレートは正常にアップデートされるはずです。
