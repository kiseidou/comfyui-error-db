### はじめに

ComfyUIを使っていて、「Add `FrontendManager` to manage non-default front-end impl」というエラーが出たことはありませんか？大丈夫、この記事ではそんなあなたの悩みを解決します！このエラーは非デフォルトのフロントエンド実装を使用する場合によく起こります。具体的な対処法を見ていきましょう。

### 前提条件

- このガイドは Windows 環境でPythonがインストールされている状態を想定しています。
- また、ComfyUI の最新版がインストール済みであることを確認してください。

### 原因の解説

このエラーは、特定のバージョンのフロントエンド（Frontend）を使用したい場合に発生します。ComfyUIでは、デフォルト以外のフロントエンドを指定する際、「Custom Node（拡張機能）」を使用することで解決できます。

具体的には、`--front-end-version`というコマンドライン引数を使って、特定のバージョンのフロントエンドを指定します。この引数はGitHubリポジトリからそのバージョンのフロントエンドをダウンロードし、それを使用する仕組みになっています。

しかし、インターネット接続がない場合や特別な状況では、この方法がうまくいかないことがあります。そのため、`--front-end-root` 引数を使用してローカルに保存したバージョンのフロントエンドを指定することも可能です。

### 解決ステップ (Step-by-Step)

#### Step 1: ログを確認する

まず、エラーが出ている状況下でのログを確認してください。特に以下のようなメッセージが表示されていないかチェックします：
```
Add `FrontendManager` to manage non-default front-end impl
```

このメッセージは、指定したフロントエンドバージョンまたはパスが正しいかどうかをチェックするためのものです。

#### Step 2: 非デフォルトフロントエンドのバージョンを設定する

次に、特定のバージョンのフロントエンドを使用する場合のコマンドライン引数を指定します。以下の例は最新版を使用する方法です：
```
--front-end-version Comfy-Org/ComfyUI_frontend@latest
```

もし特定のリリース版（例えば1.0.0）を使用したい場合は、以下のように設定してください：
```
--front-end-version Comfy-Org/ComfyUI_frontend@1.0.0
```

インターネット接続がない場合や特別な状況では、ローカルに保存したバージョンを指定します。その際は `--front-end-root` 引数を使用します：
```
--front-end-root {front-end checkout path}
```

このコマンドを実行することで、問題の原因がわかるかと思います。

### よくある質問 (FAQ)

- **インターネット接続がない場合どうすれば良いですか？**
  ローカルに保存したフロントエンドバージョンを使用するために `--front-end-root` 引数を使うことで解決します。例えば：
  ```
  --front-end-root D:\path\to\local\checkout
  ```
