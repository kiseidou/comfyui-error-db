### 【本文構成】

#### はじめに

「ComfyUIを使っていて、画面が急に表示されなくなった経験はありませんか？ エラーが出て困っていませんか？」とおっしゃる方もいるでしょう。この記事では、「Add support for simple tooltips」のエラーが出たときにどのように対処すればいいのか、初心者の方でも確実に対応できるようにステップバイステップで解説します。

#### 前提条件

このガイドは、WindowsやPython環境（仮想環境を準備している場合）を使っている方を対象にしています。ComfyUIの最新バージョンがインストールされていることを確認してください。

#### 原因の解説

「Add support for simple tooltips」エラーは、ComfyUIでカスタムノード（拡張機能）を追加しようとした際に発生するものです。このエラーは、Python環境やシステム設定に問題がある場合に起こります。

具体的には、Pythonのバージョンが古い、または必要とするライブラリがインストールされていないことが原因で起こることが多いです。また、ComfyUIの依存関係が最新ではないと、新しい機能（この場合はツールチップ）をサポートできなくなることがあります。

#### 解決ステップ (Step-by-Step)

##### ### Step 1: ComfyUIのバージョン確認する

まず、現在インストールされているComfyUIのバージョンを確認します。コマンドプロンプトやターミナルで以下のコマンドを実行してみてください。

```bash
python -m comfyui --version
```

##### ### Step 2: 必要なライブラリをインストールする

次に、ComfyUIの動作に必要なライブラリが全てインストールされているか確認します。これを行うには、仮想環境（venv）を使用して次のコマンドを実行してください。

```bash
pip install -r requirements.txt
```

もし`requirements.txt`ファイルがない場合や、エラーが出る場合は、ComfyUIのGitHubリポジトリから最新のバージョンをダウンロードし直すことをお勧めします。

##### ### Step 3: ComfyUIを更新する

最後に、ComfyUIを最新版にアップデートしましょう。これは次のコマンドで行います：

```bash
git pull origin main
```

これで、新しいツールチップ機能も一緒にインストールされます。

  A: ライブラリや依存関係が不足している場合が多いです。必要なファイルを再度ダウンロードしたり、Pythonのアップデートを行ってみてください。
