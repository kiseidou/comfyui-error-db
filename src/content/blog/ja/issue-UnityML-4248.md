### はじめに

あなたはComfyUIを使ってクリエイティブな仕事をしているものの、プログラミングには詳しくない、という経験のある方も多いでしょう。特に「Add model downloading endpoint.」と表示されるエラーが出た際は戸惑うことが多いと思います。

この記事では、そんな読者の皆さんが安心してComfyUIを操作できるように、「Add model downloading endpoint.」の原因と対処法について詳しく解説します。難しい技術用語はカッコ書きで補足し、手順も具体的に書くので、自信を持って進めていきましょう！

### 前提条件

この記事は、Windows環境を想定していますが、macOSやLinuxでも大体の手順は通用します。またPython環境があること（venv を作成済み）と、基本的なコマンドライン操作ができることが前提です。

### 原因の解説

エラー「Add model downloading endpoint.」が出る原因としては、「ComfyUIが新しい機能であるモデルダウンロード用エンドポイントを正しく設定できていない」場合が多いです。このエンドポイントは、ワークフローJSONファイルに記載されたモデルを自動的にダウンロードするためのもので（Custom Nodeと関連がある）、これを実装しないと必要なリソースが不足してしまう可能性があります。

### 解決ステップ (Step-by-Step)

#### Step 1: エンドポイントが存在するか確認する

まず、`ComfyUI`プロジェクトフォルダ内で、関連のファイルを開いてみてください。特に以下のファイルをチェックします：

```
/api/download_model.py
```

このファイルには「モデルダウンロードエンドポイント」のコードがあるはずです。

#### Step 2: コマンドを実行する

以下のコマンドを順番に実行して、必要なライブラリや設定ファイルを更新します。それぞれの手順が終わったら、次のステップへ進んでください。

1. プロジェクトフォルダ内からPython仮想環境（venv）をアクティブ化し、以下のコマンドを入力してください：

```bash
pip install -r requirements.txt
```

2. 次に、ComfyUIの設定ファイルを開き、「ダウンロードエンドポイント」が有効になっていることを確認します。例えば：
   
   ```
   # コンフィグファイル (config.yamlなど)
   download_endpoint_enabled: true
   ```

3. さらに、以下のコマンドを実行してサーバーを再起動します：

```bash
python app.py
```
