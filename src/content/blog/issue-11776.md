## エラーの概要

ComfyUIを使用中に以下のエラーメッセージが表示される場合がある。
```
comfyui-embedded-docs はバージョン 0.4.0 を使用する必要があります。現在インストールされているのは 0.3.1 のため、更新が必要です。
```

この問題はPythonの依存関係や、ComfyUI環境が最新版に対応していない時に発生します。

## 原因

- `comfyui-embedded-docs`パッケージがバージョン 0.4.0 に更新されたのに、現行のPython環境はまだ 0.3.1 をインストールしている。
- 更新されたバージョンでは、新しい言語サポート（pt-BR）や欠落していたノードドキュメンテーションなどが追加されている。

## 解決策

### Python環境を更新する手順

1. **Pythonの依存関係リポジトリを開く**
   - 環境で使用しているPythonスクリプトファイル（通常は`requirements.txt`）を見つける。

2. **dependencies.txtを編集してcomfyui-embedded-docsを更新する**
   ```diff
   - comfyui-embedded-docs==0.3.1
   + comfyui-embedded-docs==0.4.0
   ```

3. **依存関係のインストールを実行する**
   - テーマに応じて、以下のようなコマンドを使用します。
     ```sh
     pip install -r requirements.txt
     ```
   これにより、ComfyUIのPython環境が最新版に対応した状態になります。

4. **ComfyUIを再起動する**
   - Python環境の更新が完了したら、ComfyUIを完全に終了してから再度立ち上げます。
   ```sh
   python run_comfy.py
   ```

### カスタムノードの不具合がある場合

1. **カスタムノードを確認する**
   - ComfyUIで使用しているカスタムノードが最新版と互換性がない可能性があります。`Node Custom (カスタムノード)` を参照してください。

2. **互換性の問題がある場合はアップデートまたは修正する**
   - カスタムノードが更新可能な場合、最新バージョンをインストールします。
   - 互換性の問題が解決できない場合は、開発者にフィードバックを提供して解決策を探します。

## まとめ

`comfyui-embedded-docs`パッケージのアップデートは、新しい言語サポートやドキュメンテーション追加などの機能強化とともに、Python環境全体への影響も考慮する必要があります。適切な手順で依存関係を更新し、ComfyUIを再起動することで問題を解決できます。
