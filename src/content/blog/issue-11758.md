## エラーの概要

この記事は、ComfyUIで「installed models not found」（インストールされたモデルが見つからない）というエラーメッセージに遭遇したユーザーを対象としています。これは、既定のパス以外にインストールされた場合やカスタムノードを使用している場合によく発生します。

## 原因

この問題は主に以下の原因で発生します：
1. **Python環境設定不備**：ComfyUIが想定するPython環境が適切でない場合、モデルのロード時にエラーが発生します。
2. **カスタムノードのインストール先**：カスタムノードが正しく設定されていない場合、モデルパスが見つからないとエラーとなります。

## 解決策

### 1. Python環境を確認する
Pythonが適切にセットアップされているか確認します。ComfyUIはPython 3.8以上のバージョンが必要です。
- 確認コマンド:
```powershell
python --version
```
必要であれば、別のPythonバージョンを使用するために`virtualenv`や`conda`をインストールし、適切なバーチャル環境を作成します。

### 2. モデルのパスを再設定する

ComfyUIは特定のディレクトリにモデルファイルが存在することを期待しています。以下のようなディレクトリ構造を使用する必要があります：
```
C:\Users\<ユーザー名>\Documents\ComfyUI\models
```

もし、このパスが異なる場合、以下の手順で修正します。

- **extra_models_config.yaml** ファイルの編集:
  以下のように`extra-model-paths-config`パラメータに正しいパスを設定します。
```yaml
custom_nodes_path: C:\Users\<ユーザー名>\Documents\ComfyUI\custom_nodes
model_base_path: C:\Users\<ユーザー名>\Documents\ComfyUI\models
```

- **実行時の引数の修正**:
  ComfyUIの起動時に以下のようにパスを指定します。
```powershell
main.py --extra-model-paths-config C:\Users\<ユーザー名>\Documents\ComfyUI\extra_models_config.yaml
```

### 3. カスタムノードのチェックと更新

カスタムノードが正しくインストールされているか確認します。以下の手順で行います。

1. **カスタムノードのディレクトリを確認**：
   `C:\Users\<ユーザー名>\Documents\ComfyUI\custom_nodes` ディレクトリに、必要なカスタムノードが存在しているか確認します。
2. **必要であれば更新**：
   最新版のカスタムノードをGitHubから取得し、既存のディレクトリに上書きします。

## まとめ

「installed models not found」エラーは主にPython環境設定やモデルパスの誤りによるものです。適切なPythonバージョンを使用し、モデルファイルが正しいパスにあることを確認することで解決できます。カスタムノードの問題がある場合も同様に対処します。

これらの手順を踏んで解決できなければ、ComfyUIの公式ディスカッションフォーラムやGitHub Issuesで助けを求めることをおすすめします。
